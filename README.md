## 🎯 赛题背景

在光网络中，当某条传输路径（大路径）发生故障（死亡）时，其承载的业务需要被快速迁移至备用路径，以保证业务连续性。本项目针对该场景，设计了一套基于 **Yen's K-Shortest Paths** 的重路由算法，在考虑**小通道（sub-channel）占用约束**和**波长一致性**的前提下，为死亡业务寻找可行的替代路径。

## 🏗️ 核心设计

### 数据结构

| 结构体 | 字段 | 说明 |
|:---|:---|:---|
| **Edge（边/大路径）** | `id` | 边的唯一标识 |
| | `u`, `v` | 两端节点编号 |
| | `business_ids` | 途经该边的业务 ID 列表 |
| | `occupied_channels` | 已被占用的小通道集合 |
| | `available_channels` | 可用通道数 |
| | `status` | 工作状态（正常/故障） |
| **Business（业务）** | `id` | 业务唯一标识 |
| | `value` | 业务价值（用于排序） |
| | `status` | 业务状态 |
| | `src`, `dst` | 起始节点与终止节点 |
| | `edge_count` | 经过的边数量 |
| | `node_seq` | 途经节点编号序列 |
| | `path_edges` | 途经的 Edge ID 列表 |
| | `occupied_channels` | 在各边上占用的小通道 |

> **设计要点**：由于每对节点间可能存在多条大路径，通过为每条物理路径分配独立的 `Edge ID`，实现了对同一起终点不同路径的精确区分。

### 算法流程

```text
Step 1: 获取死亡路径上的所有业务，按业务价值降序排序
Step 2: 利用 Yen's K-Shortest Paths 算法，为每个业务搜索不经过死亡路径的前 K 条候选路径
Step 3: 对每条候选路径，判断其下的小通道是否满足业务的带宽通行需求
Step 4: 将业务迁移至首个满足约束的路径；若均不满足，则该业务保持死亡状态
```

## 🔑 关键难点与解决方案

| # | 难点描述 | 解决方案 |
|:---|:---|:---|
| **1** | **业务-路径-小通道三层嵌套概念复杂**：业务映射到大路径，大路径再映射到 40 个小通道 | 通过 `Edge` 结构体封装大路径，内部维护 `occupied_channels` 位图，实现通道占用状态的 O(1) 查询 |
| **2** | **死亡业务资源滞留**：若业务未成功重路由，其占用的其他未死亡路径资源不会释放 | 在重路由逻辑中增加**资源回滚机制**：仅当新路径完全分配成功后，才释放原路径资源 |
| **3** | **同起终点多路径区分**：两点间存在多条大路径，需分别管理各自的 40 个小通道 | 每条大路径赋予独立 `Edge ID`，`Business` 中记录完整的 `path_edges` 序列，精确追踪途经的每一条大路径 |
| **4** | **波长一致性约束**：业务在多条大路径上占用的小通道 ID 必须相同（如均占用 `[1:5]`） | 先筛选可行的大路径组合，再统一检查该组合下是否存在一段连续的可用小通道 |
| **5** | **节点处波长转换**：若路径中间节点有剩余“改变次数”，允许节点前后使用不同的小通道 ID | 在通道检查时，将节点两侧的通道分配解耦，消耗 1 次改变次数即可允许两侧通道 ID 不一致 |

### 小通道判断策略

考虑到赛题对运行时间和内存的严格限制，最终采用**贪心策略**：

```cpp
// 对每条候选大路径组合，按固定策略检查通道
for (int start_ch = 1; start_ch <= max_width; ++start_ch) {
    if (区间 [start_ch, start_ch + width - 1] 在所有大路径上均空闲) {
        保留该路径组合;
        break;
    }
}
```

> 该策略以牺牲部分最优解为代价，换取了极高的执行效率，避免了全排列搜索带来的超时/超空间问题。

## 🧮 Yen's K-Shortest Paths 算法详解

Yen's 算法是基于 Dijkstra 的 K 条最短路径搜索算法，通过迭代产生偏离路径（Deviation Path）来获取次优解。

### 示例图邻接矩阵

```
      C    D    E    F    G    H
C  [ inf   3    2   inf  inf  inf ]
D  [ inf  inf  inf    4   inf  inf ]
E  [ inf   1   inf    2    3   inf ]
F  [ inf  inf  inf  inf    2    1 ]
G  [ inf  inf  inf  inf  inf    2 ]
H  [ inf  inf  inf  inf  inf  inf ]
```

**参数**：源节点 `C`，目的节点 `H`，`K = 3`。  
**符号说明**：`A[i]` 为第 `i` 条最短路径；`B` 为偏离路径候选集合。

---

### 迭代过程

#### Round 1: 初始化
通过 Dijkstra 求得最短路径：
- **A[1] = C → E → F → H**，花费 = `5`

#### Round 2: 以 A[1] 为迭代路径
对 A[1] 的每个前缀节点，将前缀到下一条边的权值设为 ∞，重新跑 Dijkstra：

1. **偏离点 C**：禁用 `C-E`，求得 **A²⁻¹ = C → D → F → H**，花费 = `8`，加入 `B`
2. **偏离点 E**：禁用 `E-F`，求得 **A²⁻² = C → E → G → H**，花费 = `7`，加入 `B`
3. **偏离点 F**：禁用 `F-H`，求得 **A²⁻³ = C → E → F → G → H**，花费 = `8`，加入 `B`

从 `B` 中选择花费最小的路径：
- **A[2] = C → E → G → H**（花费 7），从 `B` 中移除。

#### Round 3: 以 A[2] 为迭代路径
1. **偏离点 C**：禁用 `C-E`，求得 **A³⁻¹ = C → D → F → H**，但 `B` 中已存在，**丢弃**
2. **偏离点 E**：禁用 `E-G`（存在于 A[2]）和 `E-F`（存在于 A[1]），求得 **A³⁻² = C → E → D → F → H**，花费 = `8`，加入 `B`
3. **偏离点 G**：禁用 `G-H`，无可用偏离路径

从 `B` 中选择花费最小的路径（C-D-F-H、C-E-F-G-H、C-E-D-F-H 花费均为 8），按**最少节点数**作为 tie-breaker：
- **A[3] = C → D → F → H**

---

### 最终结果

| 排名 | 路径 | 花费 |
|:---|:---|:---|
| 1 | C → E → F → H | 5 |
| 2 | C → E → G → H | 7 |
| 3 | C → D → F → H | 8 |

## ⚠️ 已知局限与优化方向

| 局限 | 说明 | 可能的优化 |
|:---|:---|:---|
| **次优路径风险** | Yen's 算法基于 Dijkstra 扩展，所得 K 条路径是**较优路径**，但不保证严格全局前 K 短 | 引入更精确的代价函数，或结合 CSP（Constrained Shortest Path）算法 |
| **通道判断过于粗暴** | 当前仅按 `[1 : max_width]` 顺序贪心检查，丧失部分最优解 | 改为以**小通道组合**为单位进行预筛选，减少无效的大路径判断 |
| **时间/空间瓶颈** | 遍历所有解空间会导致超时或超内存 | 采用增量式搜索（Incremental Search），或利用启发式剪枝提前排除不可行路径 |
| **缺乏通道级预过滤** | 当前以整条大路径 list 为单位判断，未提前剔除小通道明显不足的路径 | 在 Yen's 算法生成候选路径时，增加**小通道可用性**作为硬约束，直接在图搜索阶段剪枝 |
